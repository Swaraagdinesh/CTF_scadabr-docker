General

User Profiles are a new feature introduced in ScadaBR 1.1. Through a user profile, it is possible to define special levels of access to data sources, watch lists and graphical views for a set of different users, helping to manage systems with a large number of users.

Creating profiles

To add a new profile, click on the icon. A form will be displayed to allow configure the new profile.

In the Profile Name field you must define a name for your profile. This name must not be empty, or the profile will not be saved. In some versions of ScadaBR 1.1 it is possible to create more than one User Profile with the same name, however, this is an ambiguous practice and is not recommended. The EF version of ScadaBR prohibits the creation of profiles with identical names.

Below the Profile Name option, a list containing all system's data sources, watch lists and graphical views is displayed. At this list, you can define whether you want users belonging to the profile to have: no access, read access or write access to these resources (remember that write access implies read access). Specifically for data sources, there is also a checkbox to the left of the data source name which, if checked, allows read/write access to all data points in that data source.

Associating users to profiles

The association between a user and a profile must be made through the Users page. There, select the user to whom you want to associate a profile and click on his user name on the list. In the form that will open, find the User Profile option and choose the desired profile from the list. To remove a user's profile, choose the None option.

Important: each user can only be associated with a single User Profile. Administrators cannot be associated with profiles.
