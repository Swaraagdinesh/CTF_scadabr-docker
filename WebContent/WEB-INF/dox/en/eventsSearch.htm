Event search

The Event Search options allow you to search through the events in the ScadaBR database to which the user has access. If the Event Search section is collapsed, click on the icon to expand it.

In the search form, you can choose several filters for your search. The Keywords option allows you to filter specific words contained in an event, such as user notes. You can separate keywords by spaces, commas or semicolons. Each keyword is searched individually, so if you want to find an expression like "maintenance event", for example, just search for the word "maintenance". To search for keywords that should not be present at an event, add a minus sign to the beginning of the word (for example "-maintenance").

The Start date and End date options allow you to restrict your search to a range of dates. If these options are checked, you will have access to a calendar for selecting dates. Otherwise, the search will be done from the beginning and/or until the end, without filtering.

Note: In versions of ScadaBR not based on EF version, the search options by date may not exist. In this case, you must specify a value for the Max results to be displayed after the search. In this version (and in versions based on ScadaBR EF), all results will be displayed, divided into pages with up to 50 events each.

The Id option allows you to find an event with a specific Id. An Id is a unique number that identifies the event in the database. Since filtering by Id is very restrictive, it is not recommended to combine it with other filtering criteria, or the search may not return any results.

The Event source type, Alarm level and Status options allow the selection of multiple criteria. In these options, selecting all criteria and selecting none produces the same effect, that is, all criteria in the option will be included in the search. A filtering occurs when only a part of the criteria (not all) is selected.
