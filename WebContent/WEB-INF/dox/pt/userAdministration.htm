Geral

Usuários representam cada indivíduo que usa o sistema. Vários privilégios podem ser associados a usuários para garantir a eles a habilidade de ler e alterar dados e características do sistema.

Há dois níveis básicos de usuários: administradores (), e outros usuários . (). Administradores possuem acesso a todas as funcionalidade do sistema. Outros usuários possuem vários privilégios de acordo com a configuração.

Editando Usuários

Para adicionar um novo usuário, clique no ícone . O Usuário não será adicionado até ser salvo corretamente. Para editar um usuário existente, clique no nome do usuário na lista de usuários.

Senhas não são recuperáveis desde que sejam salvas; elas só podem ser resetadas especificando uma nova senha.

Para impedir que determinado usuário tenha acesso ao sistema, marque a opção "Desabilitado". Um usuário desabilitado não tem a permissão de logar no sistema. Para remover permanentemente um usuário, clique sobre o ícone de excluir ();

Para testar um endereço de email de usuário, clique no ícone . Isto fará com que um email seja enviado para o endereço de email especificado.

Usuário podem ter notificações de alarme enviadas para seu endereço de email sem explicitamente estarem especificadas em um tratador de eventos de tipo "email". Selecionar na opção Enviar e-mails de alarme (Send alarm emails) um nível de alarme acima de "nenhum" fará que essas notificações sejam enviadas quando o nível de alarme de evento for maior ou igual ao mínimo nível de alarme selecionado, porém somente quando o evento for gerado (isto é, sem incluir "aumentos" de nível ou notificação de inativos). Note também que se o usuário já estiver especificado no tratador de eventos "email", não serão enviados múltiplos emails.

Privilégio de Usuário

O checkbox Administrador determina se o usuário é um administrador.

Se o usuário não é um administrador, todos os "data sources" para o cliente serão listados. Selecionar a caixa perto do nome do "data source" indica que o usuário pode alterar a configuração do "data source", como também todos os dados produzidos por esse "data source".

Se o "data source" não está selecionado, uma lista de todos os "data points" dentro do "data source" serão mostrados. Cada "data point" pode ser individualmente setado para nenhum acesso (none), acesso de leitura (read), ou — se o "data point" for configurável — acesso de escrita (set). (Possuir acesso a escrita implica em acesso a leitura.)

Alternativamente, desde o ScadaBR 1.1 você pode definir um conjunto de permissões para um grupo de usuários através dos "Perfis de Usuários". Neste caso, as permissões individuais para cada "data source" serão ocultadas. Além dos "data sources", os Perfis de Usuário também permitem alterar as permissões para "watch lists" e representações gráficas. Perfis de Usuários só podem ser definidos para usuários que não são administradores.
